## Tech Debt Items (Non-Blocking)

These items do not block milestone completion but should be cleaned up in a future phase:

### TD-01: Dead code — `NearbyStadiumsSection.tsx`
- Phase 23 removed the Nearby Stadiums section from the UI
- `NearbyStadiumsSection.tsx` is exported but never imported anywhere
- **Impact:** Dead file in the codebase — wasted maintenance surface
- **Suggested fix:** Delete the component file

### TD-02: Dead backend computation — `nearbyStadiums` in `getStadiumById`
- `stadium.controller.ts` still runs the Haversine distance query and returns `nearbyStadiums` on every detail page request
- `StadiumDetailPage.tsx` and `StadiumDetail` type include the field but the frontend never renders it
- **Impact:** Unnecessary DB computation per request (iterates all stadiums with lat/lng)
- **Suggested fix:** Remove the Haversine loop, `nearbyStadiums` from the response, and the `NearbyStadium` interface + `nearbyStadiums` field from `StadiumDetail` type

### TD-03: Dead backend query — `pubRecPosts` in `getStadiumById`
- Controller fetches top 5 `PUB_RECOMMENDATION` posts per request and includes them in the response
- `StadiumDetailPage.tsx` never destructures or renders `pubRecPosts`
- **Impact:** Unnecessary DB query per request
- **Suggested fix:** Remove the pubRecPosts query from the controller and the field from the `StadiumDetail` type

### TD-04: Unimplemented — FanBase → Stadium link
- `TeamWithPostCount` type declares `stadiumId: number | null` with comment noting it was intended for a Stadium Guide link
- `TeamFanBasePage.tsx` never reads this field or renders a stadium link
- **Impact:** One missing navigation entry point (users can't reach a stadium from the team's FanBase page)
- **Suggested fix:** Add a small "Stadium Guide →" link in `TeamFanBasePage.tsx` using `team.stadiumId` when non-null
