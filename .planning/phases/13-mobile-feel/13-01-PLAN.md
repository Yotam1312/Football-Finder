Create `frontend/src/components/BottomNav.tsx` with the following implementation:

1. Import `React` from 'react', `{ Link, useLocation }` from 'react-router-dom', `{ Search, Users, User }` from 'lucide-react', `{ useAuth }` from '../context/AuthContext'.

2. Define a `HIDDEN_ROUTES` array: `['/login', '/register', '/welcome']`.

3. Define a `tabs` array with 3 objects:
   - `{ label: 'Search', icon: Search, path: '/', matchPrefixes: ['/', '/results', '/match'] }`
   - `{ label: 'FanBase', icon: Users, path: '/fanbase', matchPrefixes: ['/fanbase'] }`
   - `{ label: 'Profile', icon: User, path: '/profile', matchPrefixes: ['/profile'] }`

4. Export `const BottomNav: React.FC`:
   - Call `useLocation()` to get `location.pathname`
   - Call `useAuth()` to get `user`
   - If `HIDDEN_ROUTES.includes(location.pathname)`, return `null`
   - For active tab detection: a tab is active if `location.pathname === tab.path` OR (for Search tab specifically) `location.pathname.startsWith('/results')` or `location.pathname.startsWith('/match')`, for FanBase if `location.pathname.startsWith('/fanbase')`, for Profile if `location.pathname === '/profile'`
   - For Profile tab: if `user` is null, the Link href should be `/login` instead of `/profile`
   - Render a `<nav>` with classes: `fixed bottom-0 left-0 right-0 z-50 bg-white border-t border-gray-200 md:hidden` plus `pb-[env(safe-area-inset-bottom)]` for iPhone safe area
   - Inside nav: a `<div>` with `flex justify-around items-center h-14`
   - Each tab: `<Link>` to the tab path, with `flex flex-col items-center justify-center min-h-[48px] flex-1 gap-0.5`
   - Icon: `<tab.icon className="w-5 h-5" />`
   - Label: `<span className="text-xs">`
   - Active state: text and icon use `text-green-600 font-medium`, inactive use `text-gray-500`

5. The component must NOT animate on route changes (it is rendered outside AnimatePresence).
